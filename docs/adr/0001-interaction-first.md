# ADR 0001: 인터랙션 우선 — Matt Pocock 주도, Superpowers 보조

- 날짜: 2026-06-19
- 상태: 승인됨
- 관련 Spec: `docs/superpowers/specs/2026-06-19-interaction-first-guide-design.md`

## 맥락

기존 가이드는 Superpowers를 프로세스의 척추(주인공)로, Matt Pocock을 보조로 두었다.
Superpowers는 상호검증·자동화(서브에이전트 병렬 실행, 자동 검증 게이트)에 무게가
실려 있어 사람과의 인터랙션이 줄어든다. 사용자는 사람이 매 의사결정의 주인이 되는
워크플로를 원하며, Matt Pocock의 grilling이 그 인터랙션 중심성을 대표한다.

## 결정

1. 단계 흐름의 **골격은 유지**하되, 매 단계의 의사결정 게이트를 Matt Pocock의
   grilling(사람과의 집요한 인터뷰)으로 운영한다. **주인공을 Matt Pocock으로
   역전**한다.
2. Superpowers의 서브에이전트 실행·자동 검증은 **옵트인 보조 동력**으로 강등한다.
   자동 리뷰/검증은 돌리되, 그 결과를 grilling으로 사람에게 가져와 확인받은 뒤에만
   진행한다(산출물 = '제안').
3. **새 스킬을 설치하지 않고** 기존 `using-git-worktrees` 기반의 병렬 개발
   워크플로를 가이드 문서에 추가한다. 병렬 자동화는 옵트인이며 결과는 사람이 수렴한다.

## 대안

- **Superpowers 주도 유지**: 자동화·검증은 강하지만 인터랙션이 부족 — 거부.
- **단계 전면 재배열(인터랙션 우선 모델)**: 과감하나 검증된 골격을 잃음 — 거부.
- **외부 병렬 worktree 스킬 신규 설치**: 의존성·유지보수 증가, 현 요구엔 불필요 — 거부.

## 결과

- `README.md`와 `AGENTS.md`가 "사람 인터랙션 우선, 자동화는 옵트인 제안"으로 정렬됨.
- 자동 검증·코드리뷰는 사람 확인 게이트(grilling) 없이는 "완료"로 진행하지 못함.
- `skills-lock.json`은 변경되지 않는다.
